

Веб-приложение (API) для автоматизации процессов регистрации пассажиров, ведения расписания авиарейсов и управления посадочными талонами. Проект спроектирован по многослойной архитектуре с использованием современных практик разработки на платформе .NET.

#

* **Платформа:** .NET 8
* **Веб-фреймворк:** ASP.NET Core Minimal APIs
* **Взаимодействие с БД (ORM):** Entity Framework Core
* **База данных:** PostgreSQL (Провайдер Npgsql)
* **Документация и тестирование:** Swagger UI
* **Контейнеризация СУБД:** Docker

#

Приложение разделено на четкие логические слои, обеспечивающие слабую связанность компонентов:

1. **Доменный слой (model):** Содержит бизнес-сущности системы (`Flight`, `Passenger`, `Ticket`) и перечисления (`FlightStatus`), которые отображаются на таблицы базы данных.
2. **Слой доступа к данным (database):** Представлен классом `AirportDbContext`. Настраивает ограничения схемы данных (Primary Keys, NOT NULL, MaxLength) посредством Fluent API.
3. **Слой контрактов (dto):** Data Transfer Objects для разделения внутренней структуры БД и внешнего интерфейса. Включает Request/Response рекорды с первичными конструкторами.
4. **Слой бизнес-логики (services):** Сервисы (`FlightService`, `PassengerService`, `CheckInService`), реализующие ключевые правила системы.
5. **Слой инфраструктуры и маршрутизации (api):** Модули эндпоинтов для обработки HTTP-запросов и интеграция компонентов в файле `Program.cs` через Dependency Injection (IoC-контейнер).

#

В сервисе регистрации (`CheckInService`) реализованы следующие алгоритмы:

* **Автоназначение мест:** Если при регистрации пассажир не указывает желаемое место, система автоматически находит первое свободное кресло на основе массива доступных мест рейса и уже выданных билетов.
* **Валидация статусов:** Запрещена регистрация на рейсы, которые уже вылетели или были отменены.
* **Защита от дублирования:** Система проверяет, не зарегистрирован ли пассажир на данный рейс повторно.
* **VIP-овербукинг:** Пассажирам с установленным VIP-статусом разрешено проходить регистрацию даже при полной заполненности самолета. В этом случае алгоритм генерирует уникальное виртуальное VIP-место вне стандартной схемы салона.
